Multi-Agent Multi-Robot Control

LLM-driven task planning framework coordinating heterogeneous robot fleets through natural language commands

[Architecture Diagram / Demo GIF Here]

Overview

Led development of a multi-agent architecture enabling natural language control of robot fleets at Tata Elxsi. The system integrates LangChain with ROS2 to decompose high-level tasks into robot-specific actions, enabling non-technical operators to coordinate complex multi-robot workflows through conversational interfaces.

The system was validated across UR5 manipulators, Husky unmanned ground vehicles, and KUKA robotic arms in Isaac Sim, demonstrating scalable coordination and robust plan execution in both simulated and real-world environments.

The Problem

Traditional multi-robot systems require manual task allocation and hard-coded coordination logic. Scaling to heterogeneous fleets—where different robot types have different capabilities—becomes exponentially complex. Engineers must write custom code for each task variation, making rapid prototyping impossible.

Additionally, non-technical operators cannot interact with these systems, creating a bottleneck where only robotics experts can specify tasks.

Key Contributions

  • Architected LangChain-ROS2 bridge for agentic task allocation across heterogeneous fleets
  • Designed multi-agent coordination system using distributed decision-making
  • Validated system across multiple platforms: UR5 manipulator, 2× Husky UGVs, 4× KUKA arms
  • Migrated simulation environment from Gazebo to Isaac Sim for photorealistic testing
  • Scaled system from 2-robot prototype to 6+ unit deployment
  • Grew and mentored engineering team from 2 to 4 members

Technical Approach

The system uses a hierarchical planning architecture where the LLM acts as a high-level task planner, decomposing natural language commands into structured sub-tasks. These sub-tasks are then allocated to individual robots based on their capabilities and current state.

Key technical decisions:

  • LangChain for flexible LLM integration and tool use
  • ROS2 for real-time robot control and message passing
  • Docker + gRPC for microservices architecture (decoupled perception, planning, execution)
  • Isaac Sim for high-fidelity simulation with realistic physics

The system implements robust failure recovery through dynamic replanning—when a robot fails to complete a subtask, the LLM regenerates the plan and reallocates work to available robots.

Results & Impact

  • Reduced task specification time from hours (manual programming) to seconds (natural language)
  • Successfully coordinated up to 7 robots simultaneously in simulation
  • Enabled non-technical operators to deploy complex multi-robot tasks
  • System demonstrated in both controlled environments and real-world scenarios
  • Architecture patterns reused across multiple robotics projects at Tata Elxsi
